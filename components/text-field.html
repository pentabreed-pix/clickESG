Text Field

사용자가 키보드로 사용자 지정 텍스트 항목을 입력할 수 있습니다.
다양한 옵션을 필드에 표시하여 입력 요구 사항을 전달할 수 있습니다.

Anatomy

Options

Label

사용자가 입력해야 하는 세부정보를 표시합니다.
레이블 값은 컨텍스트가 충분하고 접근성 전문가가 디자인을 검토한 드문 경우 레이블이 정의되지 않을 수 있습니다.

Value

입력하기 전에 플레이스 홀더가 노출되고, 사용자가 입력한 세부 정보는 입력 중과 입력 후에 표시됩니다.

Error

오류 메시지에 수정에 대한 지침이 표시됩니다.
오류 메시지는 수정될 때까지 입력 줄 아래에 표시됩니다.

Info

사용자가 텍스트 필드에 입력해야 하는 세부 정보에 대한 추가 설명이 필요한 경우 사용할 수 있는 영역입니다.

Close icon

입력 값을 한번에 지울 수 있는 아이콘입니다.

Text icon

상태 값에 따라 변경되는 텍스트 버튼입니다.
중요도에 따라서 버튼 컬러 값으로 강조할 수 있습니다.

Focus on

입력을 위해 탭하여 텍스트 필드를 활성화한 상태입니다.

Typing

텍스트 필드를 활성화한 후 입력을 입력합니다.

Disabled

입력 불가능한 텍스트 필드입니다.

Disabled (Read only)

입력 불가능한 텍스트 필드로 고정값이 노출됩니다.
복사나 드래그는 가능하지만 상호 작용이나 텍스트 입력, 변경은 불가능한 상태입니다.

Text area

텍스트 영역은 오버플로 텍스트를 새 줄로 래핑 합니다.
고정된 높이이며 커서가 필드 하단에 도달하면 스크롤 됩니다.

Security number (Trailing ON / OFF)

사용자가 버튼을 탭하면 입력한 세부 정보를 볼 수 있으며, 다시 탭하면 숨겨진 상태로 돌아갑니다.

Combination (Drop down)

필요한 경우 다른 컴포넌트와 조합해 사용할 수 있습니다.
드롭다운 화살표는 텍스트 필드에 중첩된 선택 구성 요소가 있음을 나타냅니다.

Combination (Box button)

필요한 경우 다른 컴포넌트와 조합해 사용할 수 있습니다.
버튼 레이블은 명확하고 간결하게 작성합니다.

Usage

플레이스홀더 사용
  • 플레이스홀더는 입력 필드 내부에 간략하게 표시되는 문구입니다.
  • 필드에 입력해야 하는 세부 정보나 입력을 위한 참고 세부 정보를 간략하게 표시하여 사용자가 쉽게 정보를 입력할 수 있도록 안내하는 역할을 할 수 있습니다.

  • 레이블을 사용하지 않고 플레이스홀더로만 안내하는 경우 세부 정보를 더 명확하게 입력하는 것이 좋습니다.

  • 가능하면 레이블과 플레이스홀더에 겹치는 세부 정보를 삽입하지 마십시오. 텍스트 양이 두 배가 되어 입력하기 복잡하고 어려울 것입니다. 공간을 채우기 위한 플레이스홀더는 불필요합니다.

입력 필드 내에서 스크롤

  • 입력 필드가 확장되지 않은 한 줄 입력 항목의 경우 입력된 내용의 양에 따라 가로 스크롤이 생성될 수 있습니다.

  • 입력 필드 영역은 입력한 문자 수에 따라 수평으로 스크롤될 수 있습니다.

  • 여러 열을 허용하는 입력 필드의 경우, 입력한 문자 수가 입력 필드의 오른쪽 끝에 도달하면 자동으로 줄 바꿈이 적용됩니다.

필수 표시 사용
  • 사용자에게 정보를 요청할 때 필수로 작성해야 하는 항목에는 라벨의 오른쪽에 빨간색 * 표시를 할 수 있습니다.
  • 이 표시를 사용하면 사용자가 입력을 놓치는 것을 방지할 수 있으며, 사용자가 정보를 입력하지 않고 진행할 경우 빨간색 밑줄과 도우미 데스크를 통해
    입력이 필요하다는 것을 알릴 수도 있습니다.
