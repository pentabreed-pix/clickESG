Dropdown

드롭다운은 피커(Picker)라고 불리기도 하며 풀다운 버튼이나 아이콘 버튼과 같은 구성 요소와 함께 사용됩니다.
사용자가 목록에서 원하는 선택 항목만 필터링할 수 있도록 해줍니다.

Anatomy

Options

Label

맥락이 충분하고 접근성 전문가가 디자인을 검토한 드문
경우 레이블이 정의되지 않을 수 있습니다.

Placeholder

플레이스홀더 텍스트는 일반적으로 "고스트 텍스트"라고도 하며, 사용자에게 피커 메뉴에서 옵션을 선택하도록 요청합니다. 사용자가 옵션을 선택하면 사라집니다.

Value

값은 사용자가 선택한 옵션을 보여줍니다.

Width

너비는 상황에 맞게 적절히 사용자 지정할 수 있습니다.

Size

가이드에 있는 L, M,S 사이즈를 권장하나,
텍스트 레이블에 따라 달라질 수 있습니다.

Quiet

조용한 피커는 눈에 보이는 배경이 없을 수 있습니다.
이 스타일은 명확한 레이아웃(수직 스택, 테이블, 그리드)이 버튼을 쉽게 분석할 수 있을 때 가장 잘 작동합니다.

Menu container

모바일에서는 메뉴가 다음 중 하나로 노출될 수 있습니다.
Popover 또는 Tray로 선택할 수 있는 옵션을 사용합니다.

Required or optional

피커는 상황에 따라 선택 사항 또는 필수 사항으로 표시할 수 있습니다.

필수 레이블은 별표로 표시합니다. 선택 사항 피커는 레이블 끝에 ‘(선택)’을 표시하거나 전혀 표시되지 않습니다.

Disabled

비활성화된 상태의 피커는 입력 필드가 있지만 해당 상황에서는 사용할 수 없음을 보여줍니다.

레이아웃 연속성을 유지하고 나중에 사용할 수 있음을 전달하는 데 사용할 수 있습니다.

Info

설명은 선택해야 할 내용에 대한 구체적인 요구 사항과 같은 힌트나 유용한 정보를 전달합니다.

Popover

팝오버는 메뉴, 옵션, 추가 작업 등과 같은 일시적인 콘텐츠를 표시하는 데 사용되는 컨테이너입니다.
획과 그림자를 통해 시각적으로 눈에 띄고 인터페이스 위에 떠 있습니다.

Tray

트레이는 메뉴, 옵션, 추가 작업 등과 같은 일시적인 콘텐츠를 표시하는 컨테이너입니다.
모바일 환경에서만 존재하며 너무 압도적일 수 있는 콘텐츠 유형을 노출하는 데 사용됩니다.

Behaviors

Overflow

필드 레이블과 메뉴 텍스트가 사용 가능한 수평 공간에 비해 너무 길면 줄바꿈하여 다른 줄을 형성합니다.

필드 텍스트 자체는 끝에서 잘리지만, 텍스트는 메뉴에서 전체로 표시할 수 있습니다.

Menu height

피커 메뉴는 사용 가능한 공간에 가능한 한 많은 옵션을 표시하기 위해 필요한 만큼 높이를 정할 수 있습니다.

사용자 편의와 해상도 대응을 위해서 옵션이 지정 높이 값을 넘어갈 경우에는 스크롤을 생성합니다.

Usage

  • 화면 전체에서 동일한 입력 컨테이너 유형을 사용합니다.

  • 윤곽이 있는 유형과 포함된 유형을 함께 사용할 때 비활성화된 상태의 컨테이너가 비슷하게 보일 수 있습니다.

Internationalization

RTL(오른쪽에서 왼쪽으로) 언어의 경우 텍스트와 체크 표시는 오른쪽 정렬되고 셰브론은 왼쪽 정렬됩니다.
